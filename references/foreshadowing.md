# 伏笔设计参考

## 伏笔的三个层次

### 浅伏笔（读者读后即忘）
- 只出现一次
- 回收时：读者回忆起来，"原来如此"
- 适用：次要线索、场景道具

### 中伏笔（读者隐约有印象）
- 出现2-3次
- 每次出现都有合理表面理由
- 回收时：读者"我就知道"的感觉
- 适用：人物命运、重要道具

### 重伏笔（读者完全无意识）
- 出现3次以上
- 每次都隐藏在日常细节中
- 回收时：全书最大情感冲击点
- 适用：核心反转、真相揭示

## 伏笔登记格式

每次埋入伏笔，必须在 tracker/foreshadowing.json 中登记：

```json
{
  "伏笔编号": "fp-001",
  "类型": "人物/道具/事件/关系",
  "层级": "浅/中/重",
  "埋入章节": 3,
  "内容摘要": "男主左手腕有一道旧伤",
  "出现记录": [
    {"章节": 3, "内容": "女主注意到但没问"},

  "回收章节": null,
  "回收方式": "",
  "文学效果": ""
}
```

## 伏笔回收技巧

### 1. 意象呼应法
伏笔：墙上挂着一幅未完成的画
回收：结尾画作完成，象征人物命运闭环

### 2. 命运反转法
伏笔：算命先生说他35岁有一道坎
回收：35岁那年，他遇到了她

### 3. 误读转正法
伏笔：读者以为A是B的证据
回收：A其实是C的证据

### 4. 延迟解释法
伏笔：某个角色的行为不合逻辑
回收：30章后揭示真正动机

## 伏笔管理规则

### 硬规则
- 每个伏笔必须有回收计划（登记时必须填写"预期回收章节"）
- 连续3章未出现任何伏笔须预警
- 未回收伏笔数 > 已埋伏笔总数×5% 须预警

### 伏笔密度
- 每5000字至少埋1个浅伏笔
- 每20000字至少埋1个中伏笔
- 全书不超过5个重伏笔（超过则分散注意力）

## 常见伏笔失败模式

1. **埋了不收**：挖坑不填，读者愤怒
2. **收得太快**：伏笔刚出现就回收，无悬念
3. **收得太晚**：读者已经忘记，完全无效果
4. **回收太直**：没有任何意外感，像做作业
5. **回收方式弱**：真相揭示后，情感冲击不如预期

## 最佳伏笔案例分析

**伏笔设计**：《了不起的盖茨比》中绿灯
- 第一次出现：第一章
- 后续出现：多次，每次加一层含义
- 最终回收：全书结尾
- 效果：绿灯从一个物象变成了一种精神象征

**伏笔设计**：《赵氏孤儿》
- 孩子存活→成长→复仇
- 所有情节都是"伏笔"，全程无闲笔
- 这是最极致的伏笔写法：全书即伏笔
