# 认知偏差包与去 AI 味（Cognitive Bias Pack）

> v2 架构的「个性锻造层」。解决 AI 写作最大的通病：所有角色用同一套通用模板发声。
> 机制：给每个角色加载「认知偏差包」，让机器在约束中迸发非套路意象。

## 一、人类作为「系统架构师」

- 人类**只设标准，不写正文**。
- 标准形如：「冷蓝色忧郁」「用童年记忆映射感官」「把疼痛翻译成天气」。
- 机器在约束中生成，而非人类逐字撰写 —— 既保人类审美主导，又释放机器产能。

## 二、认知偏差包 Schema

每个角色引用一个偏差包（存于 `settings/characters/<name>/bias_pack.json`）：

```json
{
  "pack_id": "bias_pack_男主",
  "sensory_mapping": {
    "trigger": "童年被锁阁楼",
    "mapping": "把『孤独』映射为『樟木与灰尘的气味』；把『恐惧』映射为『地板缝隙的凉』"
  },
  "decision_bias": "总先假设最坏结果，再被证据推翻",
  "speech_tells": ["句尾常带不确定升调", "用反问代替肯定"],
  "forbidden_generic": ["突然", "莫名", "眼泪夺眶而出", "心中一震"]
}
```

## 三、偏差包如何消灭同质化

- 通用模板：所有角色「心中一震」「眼泪夺眶而出」「突然明白了」。
- 偏差包约束：角色 A 的恐惧 = 地板缝隙的凉；角色 B 的恐惧 = 钟表停摆的寂静。
- 同一情绪（恐惧），因 `sensory_mapping` 不同，生成**完全不同的意象** → 角色声线天然可区分。

## 四、偏差包生成协议

1. 架构师给出角色核心感受标准（如「冷蓝色忧郁」）。
2. AI 为该角色构造 `sensory_mapping`：找一个具体的童年/身体记忆作为情绪锚点。
3. 写入 `bias_pack.json`，并在角色声线卡中引用。
4. 写作该角色视角时，强制加载其偏差包，禁止 `forbidden_generic` 词。

## 五、与 PpC 的耦合

- 偏差包 id 存于角色状态机的 `cognitive_bias_pack` 字段。
- Chapter Daemon 写某角色视角章前，先 `pull` 该角色偏差包，注入写作上下文。
- 这是「角色子进程」能发出**角色特有**声音的工程基础。
