# 节点压缩图谱（Node-Compressed Graph, NCG）

> v2 架构的「记忆中枢」。把百万字级内容压缩为可检索的图结构，
> 替代「把整本书塞进上下文」导致的溢出与崩坏。

## 一、节点（每章 ≤200 字事实）

每章结束，由 Settler 进程提取事实节点并差分存入图谱。

`tracker/ncg.json` 节点示例：

```json
{
  "id": "n-0005",
  "chapter": 5,
  "text": "男主迁至城西，得知身世；与女主关系转为敌对(0.7)。灯下旧信伏笔触发(fp-003)。",
  "tags": ["位移", "关系变更", "伏笔", "信息揭示"],
  "edges": [
    { "to": "n-0003", "rel": "因果" },
    { "to": "n-0006", "rel": "同盟" }
  ]
}
```

**节点提取规则（Settler）**：

- 每章 1 个主节点，必要时 1~2 个补充节点，单节点 ≤200 字。
- 必含三类信息：触发/埋入的伏笔、角色位移、关系变更。
- 用**差分**算法：只记录相对上一章的「增量」，不重复全量复述。

## 二、超节点聚合（防止溢出）

- 远章（默认间隔 ≥10 章）自动聚为超节点（super-node）。
- 超节点类似知识图谱的边：`同盟 / 敌对 / 因果`。
- 聚合后原细节点保留 id 引用，但不进高频上下文。

```json
{
  "id": "s-01",
  "cover": "第1-12章",
  "summary": "男主流亡线：城东→城西，与女主由暧昧转敌对，身世伏笔fp-003贯穿。",
  "edges": [{ "to": "s-02", "rel": "因果" }]
}
```

## 三、三级压缩管线

| 层级 | 名称 | 内容 | 压缩策略 |
|------|------|------|----------|
| L1 | 章摘要 | 滑窗 3 章滚动摘要 | 固定保留最近 3 章节点，更旧下沉 |
| L2 | 弧微图 | 角色快照 + 伏笔台账 | 每卷生成角色状态快照与未回收伏笔清单 |
| L3 | 卷矢量 | 指南针式方向编码 | 用「方向码」表达本卷走向，如 `推进/转折/回落` |

**中文压缩估算**：中文按 `字数 × 1.5` 估算 token（runes×1.5），每个节点 ≤200 字 ≈ 300 token，
百万字 ≈ 5000 节点，配合超节点聚合后高频上下文稳定在可控量级。

## 四、多重流程图视图（类思维导图，可导出的前进路径）

NCG 是单一真源，以下视图都是它的**投影**，作者可切换：

### 1. 时间线流（timeline）
按 `chapter` 升序排列节点，展示「故事在时间上怎么走」。

### 2. 角色视角流（character）
按角色分组节点，展示「某个角色自己的经历轨迹」——用于检查配角是否断线。

### 3. 因果伏笔流（causal）
只展示带 `伏笔`/`因果` 标签的节点与边，展示「伏笔从埋到收的路径」。

生成命令：`scripts/ncg.py view <书名> <timeline|character|causal>`。

## 五、卷矢量（指南针编码）

每卷结束生成 `compass` 字段，用极简方向码记录走向，供长距离写作纠偏：

```json
{ "volume": 2, "direction": "转折", "encoding": "V2:TURN:敌友逆转" }
```

- `推进`：主线向前，冲突累积
- `转折`：关系/局势发生结构性变化
- `回落`：情绪/节奏缓冲，为下一波蓄力
